## Project Overview

node-netmask is a Node.js library for parsing and manipulating IPv4 CIDR blocks. It supports various IP notations including dotted-quad, octal, hex, and shorthand forms.

## Commands

- **Build**: `npx tsc` (compiles TypeScript source to JS + declarations in `dist/`)
- **Test all**: `npm test` (compiles then runs mocha tests via ts-node)
- **Test single file**: `mocha --require ts-node/register tests/netmask.ts`

## Architecture

The source of truth is `lib/netmask.ts`. The `dist/` directory (JS + declarations) is generated by `tsc` and gitignored — it is only built at publish time.

Key internals:
- `ip2long(ip)` — parses IP string (supports decimal, octal `0NNN`, hex `0xNN` notations) into a 32-bit unsigned integer
- `long2ip(long)` — converts 32-bit integer back to dotted-quad string
- `parseNum(s)` — parses a single numeric string token with auto-detected base (10/8/16)
- `Netmask` class — the main export; computes all network properties (base, mask, size, first, last, broadcast) in the constructor

Two test files exist in `tests/`:
- `tests/netmask.ts` — fixture table parsing, IP/block containment, forEach, hex/octal handling
- `tests/badnets.ts` — invalid input error cases

## Security Notes

This library has had CVEs related to IP parsing (CVE-2021-29418, CVE-2021-28918). Octal/hex handling in `ip2long`/`parseNum` is security-sensitive — changes to IP parsing must be tested carefully against bypass vectors.
